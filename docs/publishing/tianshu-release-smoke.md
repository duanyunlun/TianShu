# TianShu Release Smoke Baseline

This document defines the v0.5.x release-package truthfulness baseline. It exists to keep README release claims aligned with executable evidence.

## Scope

- Validate directory-style CLI archives generated by `tools/Publish-TianShuCliRelease.ps1`.
- Validate Windows, Linux, and macOS package shape through CI smoke.
- Validate `release-manifest.json` before artifacts are uploaded.
- Keep network probes out of default CI. `doctor --probe` requires real provider credentials and remains a manual/live check.

## Required Checks

1. Build release archives with `PublishSingleFile=false`, `PublishTrimmed=false`, and `SelfContained=false`.
2. Generate `release-manifest.json` with schema version, version, runtime identifiers, asset names, file sizes, SHA-256 checksums, and entry names.
3. Validate every manifest archive exists and matches its recorded size and checksum.
4. Extract each runtime package on its matching OS runner.
5. Run the packaged CLI entry with `--help`; exit code must be `0`.
6. Run `init --provider openai --config-file <temp>/tianshu.toml --cwd <temp-workspace> --json`; exit code must be `0`.
7. Confirm `init` generated `tianshu.toml`, provider instance templates, route set templates, and protocol rule templates.
8. Run `doctor --config-file <temp>/tianshu.toml --cwd <temp-workspace> --json`; exit code must be `1` when the provider API key is absent.
9. Confirm `doctor` reports `provider_api_key_missing` and does not report `packaged_assembly_missing`.
10. Confirm release packages do not contain `AGENTS.md`.

## CI Ownership

- `.github/workflows/ci-release.yml` owns release smoke for future commits and tags.
- `tools/Test-TianShuReleaseManifest.ps1` owns manifest and optional GitHub Release asset validation.
- `tools/Test-TianShuCliReleasePackage.ps1` owns package extraction and offline CLI behavior validation.

## README Claim Gate

README may say "Windows smoke-tested" only after the Windows package smoke passes.

README may say "cross-platform smoke-tested" only after Windows, Linux, and macOS package smoke pass on CI or equivalent recorded evidence.
