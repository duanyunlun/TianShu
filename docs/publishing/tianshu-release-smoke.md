# TianShu Release Smoke Baseline

This document defines the v0.5.x release-package truthfulness baseline. It exists to keep README release claims aligned with executable evidence.

## Scope

- Validate portable TianShuHome archives generated by `tools/Publish-TianShuCliRelease.ps1`.
- Validate Windows, Linux, and macOS package shape through CI smoke.
- Validate `release-manifest.json` before artifacts are uploaded.
- Keep network probes out of default CI. `doctor --probe` requires real provider credentials and remains a manual/live check.

## Required Checks

1. Build release archives with `PublishSingleFile=false`, `PublishTrimmed=false`, and `SelfContained=true`.
2. Generate `release-manifest.json` with schema version, version, runtime identifiers, asset names, file sizes, SHA-256 checksums, `layout=portable-tianshu-home`, `entryPath`, `configPath`, `modulesPath`, and `appHostPath`.
3. Validate every manifest archive exists and matches its recorded size and checksum.
4. Run `tools/Test-TianShuV06ModuleReleaseGate.ps1` to prove module templates and samples build/test, and public module docs do not contain private paths or secret literals.
5. Run `tools/Test-TianShuV07CapabilityReleaseGate.ps1` to prove the v0.7 capability surface is default-safe, write / shell / MCP mutations require human gate coverage, Memory retrieve/form/supersede enter governed RuntimeStep paths, and deterministic reactive-loop smoke is reproducible.
6. Run `tools/Test-TianShuV08RemoteContinuityReleaseGate.ps1` to prove Remote Module remains optional and explicitly activated, loopback transports do not allow public network by default, remote commands enter Host Gateway / Control Plane, remote approvals preserve human gate semantics, reconnect/idempotency/expired approval results are structured, and outbound remote projections are redacted.
7. Run `tools/Test-TianShuV09MultiAgentReleaseGate.ps1` to prove the v0.9 multi-agent release gate: `spawn_agent` is hidden by default, explicit `--enable-subagents` requires `--approve-all`, governed SubAgent tool surface only opens under module authorization, deterministic multi-agent fanout/fan-in evidence is complete, and final acceptance exposes a public observation report schema.
8. Run `tools/Test-TianShuV10DocumentationReleaseGate.ps1` to prove the v1.0 production documentation gate: README, quickstart, module guide, architecture spec, troubleshooting, security model, and release notes are present, linked, CI-wired, and free of obvious private paths or secret literals.
9. Run `tools/Test-TianShuPublicReleaseSafetyScan.ps1` to prove the public release safety scan: release-facing docs/CI/scripts do not contain real secret literals, private local paths, codex clipboard paths, tracked runtime state, tracked test artifacts, or README/docs relative dead links.
10. Run `tools/Test-TianShuReleaseAcceptanceGate.ps1` to prove tag publishing, release asset naming, manifest/checksum validation, Windows smoke, upgrade instructions, uninstall instructions, release notes, and CI wiring stay aligned.
11. Run `tools/Test-TianShuCrossPlatformCliSourceSmoke.ps1` on Windows, Linux, and macOS to prove source restore/build plus `init` and offline `doctor` smoke on each OS.
12. Extract each runtime package on its matching OS runner.
13. From a non-package CWD with `TIANSHU_HOME` cleared, run the packaged CLI entry under `bin/` with `--help`; exit code must be `0`.
14. Without running `init`, run `doctor --cwd <temp-workspace> --json`; exit code must be `1` when the provider API key is absent.
15. Confirm `doctor` reports `portableMode=true`, package-root `tianshu.toml`, writable package runtime root, packaged AppHost, `provider_api_key_missing`, and module discovered/registered counts.
16. Confirm `doctor` does not report `packaged_assembly_missing` or `apphost_missing`.
17. Run `init --provider openai --force --cwd <temp-workspace> --json`; exit code must be `0`, and config/templates must remain under the package root.
18. Run `send --message ping --kernel-runtime-loop --json --cwd <temp-workspace>` with no provider key; it may fail, but default artifacts must be under package-root `runtime/`.
19. Confirm smoke workspace does not contain `.tianshu/` or `.tianshu-cli/`.
20. Confirm release packages do not contain `AGENTS.md`.

## CI Ownership

- `.github/workflows/ci-release.yml` owns release smoke for future commits and tags.
- `tools/Test-TianShuV06ModuleReleaseGate.ps1` owns the v0.6 module SDK template/sample and public-documentation gate.
- `tools/Test-TianShuV07CapabilityReleaseGate.ps1` owns the v0.7 governed capability-surface gate for default safety, human gate coverage, Memory capability routing, context policy execution, and deterministic reactive-loop smoke.
- `tools/Test-TianShuV08RemoteContinuityReleaseGate.ps1` owns the v0.8 remote continuity gate for default-off remote module activation, non-public loopback transport defaults, governed remote command ingress, remote projection redaction, reconnect/idempotency semantics, and deterministic remote continuity acceptance evidence.
- `tools/Test-TianShuV09MultiAgentReleaseGate.ps1` owns the v0.9 multi-agent release gate for default-limited SubAgent exposure, explicit approval-bound opt-in, deterministic fanout/fan-in mechanism evidence, and public live-observation report schema checks.
- `tools/Test-TianShuV10DocumentationReleaseGate.ps1` owns the v1.0 production documentation gate for README, quickstart, module guide, architecture spec, troubleshooting, security model, release notes, CI wiring, and basic public-document hygiene.
- `tools/Test-TianShuPublicReleaseSafetyScan.ps1` owns public release safety scanning for secret literals, private local paths, runtime state, tracked test artifacts, package `AGENTS.md` exclusion, and README/docs relative dead links.
- `tools/Test-TianShuReleaseAcceptanceGate.ps1` owns release acceptance wiring for tag-triggered publishing, asset naming, manifest/checksum contract, Windows smoke, upgrade/uninstall instructions, release notes, and CI release chain.
- `tools/Test-TianShuCrossPlatformCliSourceSmoke.ps1` owns cross-platform source restore/build/init/doctor smoke for Windows, Linux, and macOS.
- `tools/Test-TianShuReleaseManifest.ps1` owns manifest and optional GitHub Release asset validation.
- `tools/Test-TianShuCliReleasePackage.ps1` owns package extraction and offline CLI behavior validation.

## README Claim Gate

README may say "Windows smoke-tested" only after the Windows package smoke passes.

README may say "cross-platform smoke-tested" only after Windows, Linux, and macOS package smoke pass on CI or equivalent recorded evidence.
